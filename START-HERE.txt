│ 🔑 IMPORTANT: LAPTOP IP CONFIGURATION                                       │
└──────────────────────────────────────────────────────────────────────────────┘

⚠️  YOU MUST DO THIS OR IT WON'T WORK!

When you create the Android app, you'll need to update MainActivity.kt:

FIND THIS:
──────────
    private val LAPTOP_IP = "192.168.1.100"  // ← DEFAULT VALUE

REPLACE WITH YOUR IP:
─────────────────────
    private val LAPTOP_IP = "192.168.0.5"    // ← YOUR ACTUAL IP

HOW TO FIND YOUR IP:
────────────────────
Windows (PowerShell):
    ipconfig
    Look for: IPv4 Address: 192.168.x.x

Mac (Terminal):
    ifconfig | grep "inet " | grep -v 127.0.0.1
    Look for something like: 192.168.x.x

Linux (Terminal):
    hostname -I
    Look for: 192.168.x.x

VERIFY:
───────
Test from laptop browser:
    http://192.168.x.x:3000/status   (replace with your IP)

┌──────────────────────────────────────────────────────────────────────────────┐
│ 🚨 QUICK TROUBLESHOOTING                                                     │
└──────────────────────────────────────────────────────────────────────────────┘

ISSUE: "npm install" fails
SOLUTION: 
  - Check Node.js version: node --version (need v14+)
  - Delete node_modules folder and package-lock.json
  - Run npm install again

ISSUE: Port 3000 already in use
SOLUTION:
  - Close other applications using port 3000
  - Or change port in server.js (line 32)

ISSUE: Android app says "Cannot reach server"
SOLUTION:
  - Verify laptop IP is correct (run ipconfig)
  - Check both devices on same WiFi
  - Ensure server is running on laptop
  - Check Windows firewall allows port 3000

ISSUE: NFC doesn't detect card
SOLUTION:
  - Enable NFC in phone settings
  - Card must be NDEF-formatted
  - Try different card positions
  - Check phone has NFC hardware

ISSUE: Patient page blank
SOLUTION:
  - Open browser console (F12)
  - Check nfc-tester.html gets error
  - Verify patient.html is in correct folder
  - Check server logs for errors
